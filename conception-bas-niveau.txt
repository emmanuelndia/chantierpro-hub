#**Exigences :  

	- Quelles opérations ce système doit t'il prendre en charge ? : 

		. Un technicien pointe son arrivée et sa sortie sur un chantier afin de déclarer sa présence, sous réserve d'être dans un rayon de 2 km

		. Un technicien prend une photo instantanée rattachée à son chantier afin de documenter l'avancement ou un incident

		. Un technicien consulter son historique de pointages afin de vérifier ses heures déclarées

		. Un chef de projet voit en temps réel quels techniciens sont présents sur ses chantiers afin de suivre l'activité terrain	

		. Un chef de projet consulte les photos d'un chantier afin de suivre l'avancement des travaux à distance

		. Un chef de projet prend une photo lors de sa visite afin de documenter l'état du chantier

		. Un chef de projet supprime une photo inappropriée afin de maintenir la qualité de la documentation

		. Un membre de la direction a accès à tous les chantiers actifs afin d'avoir une vision globale de l'activité	

		. Un membre de la direction prend et consulte des photos sur n'importe quel chantier afin d'assurer mon suivi terrain


	- Quelles conditions définissent le succès, l'échec, ou le moment où le système s'arrête ou transite à l'état ?
		
		. Le pointage est refusé si la distance au chantier > 2 km

		. Une photo est immédiatement rattachée au chantier, horodatée et géolocalisée

		. Une photo supprimée génère un log traçable  


	- Comment le système doit-il réagir lorsque des entrées ou actions sont invalides ?

		. En cas de distance au chantier > 2 km, message d'erreur explicite avec la distance au chantier

		

	- Quels sont les domaines sont dans les champ d'application (logique de base, règles métier) et quels domaines sont explicitement exclus (interface utilisateur, stockage, réseau, concurrence, extensibilité) ?

		* champ d'application : 

			- Authentification & Gestion des comptes
			
			- Pointage Géolocalisé (Technicien)

			- Documentation Photographique

			- Gestion de Projets, Chantiers, Equipes (Chef de Projet / Direction)

			- Export CSV / Excel des données mensuelles


		* Hors champ : 

			- Couche UI/rendu - interface utilisateur

Un User peut selon le rôle :
	- créer projets/chantiers/équipes
	- pointer
	- prendre des photos 

Un Projet contient plusieurs Chantiers

	- des Pointages
	- des Photos
	- des Equipes

Une Equipe contient plusieurs Users via Equipe_membres

Une Photo appartient à: 
	- un Chantier
	- un User

Une suppression de photo par un User autorisé est dans LogPhoto

#** IMPLEMENTATION DES METHODES PRINCIPALES

. User.pointer(chantier, type, lat, lng) -> pointage

pointer(chantier, type, lat, lng):

    // Cas limites
    if not self.isActive
        raise "Compte utilisateur inactif"

    if not chantier.estActif()
        raise "Ce chantier n'est pas actif"

    if type == ARRIVEE
        if chantier.aDejaUnPointageOuvert(self)
            raise "Une session est déjà ouverte sur ce chantier"

    if type == DEPART
        if not chantier.aDejaUnPointageOuvert(self)
            raise "Aucune arrivée enregistrée pour clôturer"

    // Happy path 
    distance = calculerDistance(lat, lng, chantier.localisation)

    if distance > 2.0
        raise "Pointage refusé : vous êtes à " + distance + " km du chantier"

    pointage = new Pointage(
        chantier_id       = chantier.id,
        user_id           = self.id,
        type              = type,
        date_pointage     = aujourdhui(),
        heure_pointage    = maintenant(),
        latitude          = lat,
        longitude         = lng,
        statut            = VALIDE,
        distance_chantier = distance
    )

    return pointage

Note: User orchestre l'action, Chantier valide le contexte, Pointage encode le résultat


. Chantier.verifierPointage(user, lat, lng) -> bool

verifierPointage(user, lat, lng):

    // Cas limites 
    if not self.estActif()
        return false

    if user == null
        return false

    // Happy path 
    distance = calculerDistance(lat, lng, self.localisation)

    return distance <= 2.0



. Photo.supprimer(deletedBy, raison) -> LogPhoto

supprimer(deletedBy: User, raison: string) -> LogPhoto:

    // Cas limites 
    if not deletedBy.peutSupprimerPhoto()
        raise "Permission refusée : rôle insuffisant"

    if not deletedBy.isActive
        raise "Compte utilisateur inactif"

    if raison == null or raison == ""
        raise "Une raison est obligatoire pour la suppression"

    // Happy path 
    log = new LogPhoto(
        photo_id         = self.id,
        deleted_by       = deletedBy,
        raison           = raison,
        date_suppression = maintenant()
    )


    return log

Note: LogPhoto conserve photo_id, pour préserver la traçabilité
