ChantierPro - Donnees de test API pour Postman
Base URL: http://localhost:3000

==================================================
1. CONFIGURATION A FAIRE
==================================================

Fichier a configurer
- Creer ou completer : .env.local
- Emplacement : racine du projet
- Chemin local : C:\Users\EMMANUEL N'DIA\Desktop\chantierpro-hub\.env.local

Variables minimales requises
DATABASE_URL=postgresql://...
JWT_SECRET=une_cle_longue_et_secrete
JWT_REFRESH_SECRET=une_autre_cle_longue_et_secrete
STORAGE_URL=https://storage.example.com/chantierpro

Variables Supabase Storage recommandees
SUPABASE_URL=https://ton-projet.supabase.co
SUPABASE_SERVICE_ROLE_KEY=ta-service-role-key
SUPABASE_STORAGE_BUCKET=chantierpro-photos

Variable geocoding recommandee
MAPBOX_ACCESS_TOKEN=ton-token-mapbox-serveur

Variable utile pour les tests smoke
AUTH_SMOKE_PORT=3105

Ou chercher chaque valeur
- DATABASE_URL : dashboard Neon / PostgreSQL / fournisseur de base
- JWT_SECRET : a generer toi-meme, cle longue aleatoire
- JWT_REFRESH_SECRET : a generer toi-meme, differente de JWT_SECRET
- STORAGE_URL : valeur de base du mode mock stockage, a garder pour le fallback local
- SUPABASE_URL : dashboard Supabase > Settings > API > Project URL
- SUPABASE_SERVICE_ROLE_KEY : dashboard Supabase > Settings > API > service_role key
- SUPABASE_STORAGE_BUCKET : nom exact du bucket prive Supabase Storage
- MAPBOX_ACCESS_TOKEN : dashboard Mapbox > Access tokens > token secret utilise par le proxy backend
- AUTH_SMOKE_PORT : libre, optionnel, seulement pour scripts/auth-smoke.ts

Ou le code utilise ces variables
- prisma/schema.prisma -> DATABASE_URL
- lib/auth/tokens.ts -> JWT_SECRET, JWT_REFRESH_SECRET
- lib/storage.ts et lib/photo-storage.ts -> STORAGE_URL, SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, SUPABASE_STORAGE_BUCKET
- lib/project-web.ts -> MAPBOX_ACCESS_TOKEN pour l'autocompletion adresse chantier
- scripts/auth-smoke.ts -> AUTH_SMOKE_PORT

Important
- Redemarrer le serveur apres modification de .env.local
- Si Supabase Storage n'est pas configure, le helper photo utilise un mode mock signe

Commandes utiles apres configuration
- npm install
- npx prisma db seed
- npm run dev

==================================================
4. REGLES POSTMAN RAPIDES
==================================================

- Les routes photo utilisent Body = form-data
- Les autres payloads utilisent en general Body = raw JSON
- Pour les routes protegees : Authorization: Bearer <ACCESS_TOKEN>
- Pour /api/auth/refresh et /api/auth/logout : gerer le cookie chantierpro_refresh
- ACCESS_TOKEN_TERRAIN = SUPERVISOR ou COORDINATOR ou GENERAL_SUPERVISOR

7. NOTES IMPORTANTES
==================================================

- `DELETE /api/photos/logs/:id` doit renvoyer 403 pour tous les roles, y compris ADMIN

- `GET /api/direction/*` est reserve a DIRECTION et ADMIN
- `GET /api/rh/*` et `POST /api/rh/export` sont reserves a HR, DIRECTION, ADMIN
- Les routes terrain utilisent les roles SUPERVISOR, COORDINATOR, GENERAL_SUPERVISOR
