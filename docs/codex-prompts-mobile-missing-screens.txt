# ChantierPro - Prompts Codex Pour Ecrans Mobiles Manquants

Ce fichier regroupe les prompts a copier dans Codex pour completer les ecrans mobiles manquants ou incomplets.

Contexte commun a tous les prompts :
- Application : ChantierPro, gestion de chantiers BTP.
- Maquettes : docs/prototype_mobile et prototype_mobile.zip.
- Shell mobile existant : app/(mobile)/mobile/(dashboard)/layout.tsx, MobileAppShell, BottomTabBar, OfflineBanner.
- Respecter safe area iOS/Android, UI mobile-first, boutons tactiles min 48-56px, actions principales visibles.
- Utiliser React Query avec staleTime : 30s pour listes dynamiques, 5min pour options peu changeantes.
- Gerer loading, empty, error states.
- Corriger les textes encodes casses.
- Ne pas exposer de suppression photo sur mobile.
- Scopes roles :
  - PROJECT_MANAGER : uniquement ses projets/chantiers.
  - DIRECTION : vue globale active.
  - GENERAL_SUPERVISOR : planning, rapports, equipes selon perimetre operationnel.
  - HR exclu des ecrans management mobile.
- Verification attendue a chaque prompt : npm run lint, npm run build. Si les checks echouent sur des erreurs globales preexistantes hors perimetre, le signaler et fournir un lint cible sur les fichiers modifies.

# MOB-MGMT-1 Galerie Photos PROJECT_MANAGER / DIRECTION

## Summary
- Finaliser la galerie photo mobile management sur `/mobile/gallery`.
- Accessible uniquement a `PROJECT_MANAGER` et `DIRECTION`.
- Garder `/mobile/photo` comme route camera directe pour les roles terrain.

## Key Changes
- Route `/mobile/gallery` :
  - session absente -> `/mobile/login?next=/mobile/gallery...`;
  - `PROJECT_MANAGER`, `DIRECTION` autorises;
  - roles terrain -> `/mobile/photo`;
  - autres roles -> `/mobile/profile`.
- Navigation :
  - la tab `Photo` des `PROJECT_MANAGER` et `DIRECTION` pointe vers `/mobile/gallery`;
  - la tab `Photo` terrain reste `/mobile/photo`.
- Data :
  - charger les chantiers via `GET /api/mobile/photo/sites`;
  - charger les photos via `GET /api/sites/:id/photos?page=&sort=desc&from=&to=&uploadedBy=`;
  - `PROJECT_MANAGER` voit seulement les photos des chantiers de ses projets;
  - `DIRECTION` voit les chantiers actifs globaux.
- UI inspiree de `docs/prototype_mobile/management/galerie.html` :
  - header compact `Galerie photos`, chantier selectionne, total photos;
  - grille 2 colonnes, batch de 20, scroll infini;
  - filtres date debut/fin et auteur multi-select;
  - lightbox plein ecran avec swipe horizontal, precedent/suivant, overlay auteur/date/site;
  - mention exacte : `Suppression disponible sur l'application web`;
  - aucune action de suppression.

## Test Plan
- `npm run lint`
- `npm run build`
- Verifier `/mobile/gallery` avec `PROJECT_MANAGER` et `DIRECTION`.
- Verifier `/mobile/gallery?siteId=...`.
- Verifier filtres, pagination, lightbox, absence de suppression.
- Verifier qu'un `PROJECT_MANAGER` ne voit pas de photos hors de ses projets.

## Assumptions
- La galerie mobile management est contextualisee par chantier; si aucun `siteId`, selectionner le premier chantier accessible.
